# 야구 커뮤니티 '야!구했어?' 프로젝트

<br> 

**🧢 야구 팬들을 위한 맞춤형 커뮤니티 플랫폼**  
야구 팬들이 선수 뉴스, 구단 응원, 소모임 후기 등 다양한 정보를 공유하고 소통할 수 있는 공간을 제공합니다.

>  팬덤 문화 활성화, 실시간 소통, 굿즈 자랑까지! 야구를 사랑하는 모두의 소통 허브

<br>

## 📌 프로젝트 개요

**📅 프로젝트 기간** 
  - 2024.12.03 ~ 2025.01.06

<br>

**📝 프로젝트 목표**
  - Servlet과 JSP 기반 웹 애플리케이션 개발
  - 팬 중심의 커뮤니티 서비스 구현
  - 뉴스 크롤링 기능을 통한 실시간 뉴스 제공

<br>

**👨‍💻 담당 역할**
  - **프로젝트 팀장**
    - 프로젝트 일정 관리
    - 코드 병합

  - **로그인/로그아웃**
    -  세션 활용 로그인 정보 관리

  - **소모임 게시판**
    - 게시글 CRUD
    - 좋아요/싫어요
    - 댓글/답글 Ajax 활용 비동기 처리
    - 지역/카테고리별 비동기 필터
    - 페이징 처리

  - **크롤링 스케줄러 구현**
    - WebListener 기반 뉴스 수집 자동화(15분 간격)
    - Selenium 및 크롬 드라이브를 활용하여 스포츠 뉴스 및 이미지 크롤링
    - 파일 저장 및 DB 저장 처리


<br>

## 참여 소감

첫 프로젝트였던 만큼, 이번 세미 프로젝트를 통해 소프트웨어 개발이란 어떤 것인지 작게나마 몸으로 느낄 수 있는 경험이었습니다. 단순히 코드를 작성하는 것이 아니라, 기획부터 구현, 그리고 문제 해결까지 모든 과정을 팀원들과 협업하고 해결해 나가는 과정에서 개발에 더욱 재미를 느꼈습니다.

담당했던 크롤링 기능 구현 과정에서는 많은 시행착오를 겪었습니다. 원하는 데이터를 가져오기 위해 인프런 강의, 유튜브 영상, 블로그 등을 찾아보며 하나하나 적용해보는 과정에서 오류를 찾고 해결하는 것이 쉽지는 않았지만, 결국 목표한 기능을 구현하고 실제로 작동하는 모습을 보며 큰 성취감과 뿌듯함을 느낄 수 있었습니다.

하지만 DB 설계에서 놓친 부분과 github를 통해 형상관리를 못했던 점은 아쉬웠습니다. 프로젝트 진행 중 여러 번의 테이블 수정으로, 관련된 소스코드를 모두 수정하면서 DB 설계의 중요성을 깨달았습니다. 또한 github 사용에 어려움을 겪어 팀원들의 소스코드를 직접 병합하면서 형상관리 도구 사용 또한 개발에 필수적임을 알 수 있었습니다. 이 경험을 통해 프로젝트가 끝난 후 Github 사용법을 공부하여 팀원들에게 공유한 후 함께 실습한 결과, 다음 프로젝트에서 형상관리를 성공적으로 해낼 수 있었습니다.

이번 프로젝트를 통해 모르는 것을 스스로 찾아보고 해결해나가는 태도의 중요성을 체감할 수 있었고, 그것이 개발자로서 성장하는 데 있어 매우 중요한 능력이라는 것을 깨달았습니다. 이 경험을 바탕으로 앞으로의 프로젝트에서도 능동적으로 문제를 해결하고 싶습니다.
